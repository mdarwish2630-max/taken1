المشروع هو نظام CMS متعدد المستأجرين مبني بـ PHP مع إدارة قوالب واشتراكات مدفوعة.

=== الملف: app/config/config.php =============================================

[FIX-01] تعطيل DEBUG_MODE في بيئة الإنتاج
  - الخطورة: حرجة
  - المشكلة: DEBUG_MODE = true يعرض تفاصيل الأخطاء للمستخدم مما يسهل الهجمات
  - الإصلاح: تغيير إلى false مع تفعيل log_errors

[FIX-02] إضافة Security Headers
  - الخطورة: عالية
  - المشكلة: عدم وجود رؤوس أمان
  - الإصلاح: إضافة CSP, X-Frame-Options, X-Content-Type-Options, HSTS, 
    X-XSS-Protection, Referrer-Policy, Permissions-Policy

[FIX-03] تحسين إعدادات الجلسة
  - الخطورة: عالية
  - المشكلة: عدم التحقق من HTTP_X_FORWARDED_PROTO للكشف عن HTTPS خلف proxy
  - الإصلاح: دعم الكشف عن HTTPS عبر proxy

[FIX-04] تجديد معرف الجلسة دورياً
  - الخطورة: عالية
  - المشكلة: عدم تجديد Session ID مما يسهل Session Fixation
  - الإصلاح: session_regenerate_id عند أول استخدام وعند انتهاء الصلاحية

[FIX-05] ربط الجلسة بعنوان IP
  - الخطورة: متوسطة
  - المشكلة: يمكن سرقة الجلسة واستخدامها من IP آخر
  - الإصلاح: تخزين IP عند إنشاء الجلسة

[FIX-06] حماية متغيرات الإدخال
  - الخطورة: منخفضة
  - المشكلة: magic_quotes قد تكون مفعلة في إصدارات قديمة
  - الإصلاح: تعطيل magic_quotes_gpc

=== الملف: app/controllers/AuthController.php ================================

[FIX-07] تعزيز Rate Limiting لتسجيل الدخول
  - الخطورة: حرجة
  - المشكلة: Rate limit 5 محاولات في دقيقة واحدة فقط
  - الإصلاح: استخدام MAX_LOGIN_ATTEMPTS (5) و LOGIN_LOCKOUT_TIME (15 دقيقة)

[FIX-08] توحيد التحقق من طول كلمة المرور
  - الخطورة: عالية
  - المشكلة: updatePassword يستخدم 6 أحرف بينما التسجيل يستخدم 8
  - الإصلاح: توحيد استخدام PASSWORD_MIN_LENGTH و PASSWORD_MAX_LENGTH

[FIX-09] إضافة CSRF protection لـ resendVerification
  - الخطورة: عالية
  - المشكلة: إعادة إرسال إيميل التحقق بدون تحقق CSRF
  - الإصلاح: إضافة $this->verifyCsrf()

[FIX-10] التحقق من صلاحية Token في resetPassword
  - الخطورة: حرجة
  - المشكلة: Token غير validated - ممكن SQL Injection
  - الإصلاح: إضافة regex validation للـ token

[FIX-11] تجديد Session ID بعد العمليات الحساسة
  - الخطورة: حرجة
  - المشكلة: عدم تجديد Session ID بعد تسجيل الدخول/تغيير كلمة المرور
  - الإصلاح: إضافة session_regenerate_id(true) بعد العمليات الحساسة

[FIX-12] Rate Limiting لإعادة إرسال إيميل التحقق والتسجيل
  - الخطورة: عالية
  - المشكلة: يمكن إرسال إيميلات بشكل متكرر (Spam)
  - الإصلاح: إضافة rate limiting (2/5min للإيميل, 3/5min للتسجيل)

=== الملف: app/controllers/AdminController.php ===============================

[FIX-13] SQL Injection في subscriptions()
  - الخطورة: حرجة
  - المشكلة: $perPage و $offset مدمجة مباشرة في SQL
  - الإصلاح: استخدام ? placeholders مع array_merge للـ params

[FIX-14] Direct $_POST access في updateSettings()
  - الخطورة: عالية
  - المشكلة: استخدام $_POST[$key] مباشرة بدلاً من $this->input()
  - الإصلاح: استخدام $this->input($key) مع null coalescing

[FIX-15] كشف معلومات في رسائل الخطأ
