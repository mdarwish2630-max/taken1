=====================================================
        صلاحيات المجلدات والملفات الآمنة
=====================================================

الصلاحيات الصحيحة:

المجلدات:
---------
755 (drwxr-xr-x)
- المالك: قراءة + كتابة + تنفيذ (rwx)
- المجموعة: قراءة + تنفيذ (r-x)
- الآخرون: قراءة + تنفيذ (r-x)

الملفات:
--------
644 (rw-r--r--)
- المالك: قراءة + كتابة (rw-)
- المجموعة: قراءة فقط (r--)
- الآخرون: قراءة فقط (r--)

مجلد uploads:
------------
المجلد الرئيسي: 755
الملفات المرفوعة: 644 (يتم تعيينها تلقائياً عند الرفع)

=====================================================
         أوامر ضبط الصلاحيات
=====================================================

# ضبط صلاحيات المجلدات
find /path/to/takweenweb -type d -exec chmod 755 {} \;

# ضبط صلاحيات الملفات
find /path/to/takweenweb -type f -exec chmod 644 {} \;

# صلاحيات خاصة لمجلد uploads (للسماح بالكتابة)
chmod 755 /path/to/takweenweb/uploads

# في بيئة الإنتاج، المالك يجب أن يكون مستخدم الويب
# مثال على Apache:
chown -R www-data:www-data /path/to/takweenweb/uploads

# مثال على Nginx:
chown -R nginx:nginx /path/to/takweenweb/uploads

=====================================================
              ⚠️ تحذيرات أمنية
=====================================================

❌ لا تستخدم 777 أبداً!
   - يعطي صلاحيات كاملة للجميع
   - يمكن لأي شخص قراءة/كتابة/تنفيذ الملفات
   - عرضة لرفع ملفات خبيثة

✅ استخدم 755 للمجلدات و 644 للملفات
✅ اجعل المالك هو مستخدم الويب (www-data/nginx)
✅ استخدم .htaccess لمنع تنفيذ PHP في uploads
